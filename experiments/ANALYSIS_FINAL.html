  • Strategy A Baseline (L, K constant)
  • Strategy D Aggressive curriculum
  • Strategy E Curriculum v3 with rule K = 2L + 1

4. Plotting training loss and win rate vs random

In this cell we define three helper functions that will be used later to visualize and analyse the behaviour of the three training strategies (A, D, E).

  1. plot_loss_curves(logs: pd.DataFrame)

    • Plots the training loss (MSE) as a function of the training iteration.
    • For each strategy, it takes all its log rows and draws a curve loss(iter) with a different label:
      • Strategy A = baseline (constant L, K),
      • Strategy D = aggressive curriculum,
      • Strategy E = curriculum with constraint (K = 2L + 1).
    • This plot lets us see how fast and how stably each strategy converges.

  2. build_winrate_by_iter(logs: pd.DataFrame) -> pd.DataFrame

    • Extracts only the iterations where we actually evaluated the agent against a random opponent (i.e., where wins_vs_random, draws_vs_random, and losses_vs_random are not null).
    • For each of those rows it computes: [ \text{games} = \text{wins} + \text{draws} + \text{losses}, \quad \text{winrate} = \frac{\text{wins}}{\text{games}} ]
    • Returns a compact DataFrame with columns:
      • strategy (A / D / E),
      • iter (training iteration),
      • winrate (empirical win rate vs random at that iteration).
    • This is a preprocessing step used by the plotting function.

  3. plot_winrate_curves(df_eval: pd.DataFrame)

    • Takes the DataFrame produced by build_winrate_by_iter and plots the win rate vs random as a function of the iteration for each strategy.
    • The y-axis is in ([0, 1]), so:
      • 0.0 = always loses against random,
      • 0.5 = about as strong as random,
      • 1.0 = always wins against random.
    • This plot is more interpretable than loss, because it directly answers:

      How strong is the agent as a player after t training iterations?

def plot_loss_curves(logs: pd.DataFrame):
    """
    Plot training loss (MSE) vs iteration for each strategy.
    """
    if logs.empty:
        print("No logs to plot.")
        return

    plt.figure(figsize=(8, 5))
    for s, df_s in logs.groupby("strategy"):
        plt.plot(df_s["iter"], df_s["loss"], marker="o", label=f"Strategy {s}")
    plt.xlabel("Iteration")
    plt.ylabel("Training loss (MSE)")
    plt.title("Training loss vs iteration")
    plt.grid(True, alpha=0.3)
    plt.legend()
    plt.show()


def build_winrate_by_iter(logs: pd.DataFrame) -> pd.DataFrame:
    """
    Build a DataFrame with the win rate vs random *per iteration* for each strategy.
    Only rows with non-null wins/draws/losses are used (typically every 10 iterations).
    """
    if logs.empty:
        return pd.DataFrame()

    # Keep only rows where we actually played games vs random
    mask = logs["wins_vs_random"].notna()
    df_eval = logs[mask].copy()

    # Compute number of games and empirical win rate
    df_eval["games"] = (
        df_eval["wins_vs_random"]
        + df_eval["draws_vs_random"]
        + df_eval["losses_vs_random"]
    )
    df_eval["winrate"] = df_eval["wins_vs_random"] / df_eval["games"]

    return df_eval[["strategy", "iter", "winrate"]]


def plot_winrate_curves(df_eval: pd.DataFrame):
    """
    Plot win rate vs random as a function of iteration for each strategy.
    """
    if df_eval.empty:
        print("No winrate-vs-random data to plot.")
        return

    plt.figure(figsize=(8, 5))
    for s, df_s in df_eval.groupby("strategy"):
        plt.plot(df_s["iter"], df_s["winrate"], marker="o", label=f"Strategy {s}")
    plt.xlabel("Iteration")
    plt.ylabel("Win rate vs random")
    plt.ylim(0, 1.05)
    plt.title("Win rate vs random vs iteration")
    plt.grid(True, alpha=0.3)
    plt.legend()
    plt.show()

8. Offline evaluation helpers (MSE and correlation vs deep MinMax)

This cell defines helper functions used later to evaluate each strategy offline, i.e., without running full games.

What the code does

  • It imports:

    • generate_test_states from experiments.eval_offline_connect4, which produces a set of non-trivial Connect Four positions together with an approximate ground-truth value computed by a deep MinMax search without heuristic.
    • state_to_np_array from utils.encoding, which converts a game state into the flat numeric vector used as input to the network.

  • It redefines an MLP class:

    • Same architecture as the networks used in training:
      two hidden layers with ReLU, final Tanh output in ([-1, 1]).
    • This local definition is only used for loading checkpoints and running evaluation.

  • The function offline_eval_checkpoint(ckpt_path, n_states, seed, device):

    1. Loads a trained model from the checkpoint file ckpt_path.
    2. Generates n_states board positions via generate_test_states, which:
      • start from the initial position,
      • play a short random playout (up to 12 moves),
      • evaluate each resulting state with a deeper MinMax search ((L = 4), no heuristic), yielding a true value for that position.
    3. For each state:
      • encodes the board via state_to_np_array,
      • applies the network to obtain the predicted evaluation,
      • collects all predictions into an array preds.
    4. Computes:
      • the mean squared error (MSE) between preds and the MinMax values true_vals;
      • the Pearson correlation between preds and true_vals
        (set to 0.0 if one of the two vectors has zero variance, in which case the correlation is undefined).

What this cell is for

This cell does not produce output by itself: it only defines tools that will be used in the next cell to compare checkpoints.

Conceptually, it provides a way to answer the question:

How well does (H_{\text{true}}) approximate the outcome predicted by a deeper, more expensive MinMax search?

By looking at MSE and correlation for each strategy, we can measure how faithfully the learned evaluator reproduces the behaviour of a stronger search procedure on the same game states.

from experiments.eval_offline_connect4 import generate_test_states
from utils.encoding import state_to_np_array

class MLP(nn.Module):
    def __init__(self, input_dim: int, hidden_dim: int = 128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1),
            nn.Tanh(),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.net(x)


def offline_eval_checkpoint(ckpt_path: str,
                            n_states: int = 200,
                            seed: int = 0,
                            device: str = "cpu"):
    """
    Carica un MLP dal checkpoint `ckpt_path` e calcola MSE + correlazione
    rispetto ai valori 'veri' approssimati via MinMax (generate_test_states).
    """
    rows, cols = 6, 7
    input_dim = rows * cols

    if not os.path.exists(ckpt_path):
        print(f"[WARN] checkpoint non trovato: {ckpt_path}")
        return None, None

    model = MLP(input_dim=input_dim, hidden_dim=128)
    state_dict = torch.load(ckpt_path, map_location=device)
    model.load_state_dict(state_dict)
    model.to(device)
    model.eval()

    states, true_vals = generate_test_states(
        n=n_states,
        max_random_moves=12,
        seed=seed
    )

    preds = []
    with torch.no_grad():
        for s in states:
            x_np = state_to_np_array(s)
            x = torch.tensor(x_np, dtype=torch.float32, device=device).unsqueeze(0)
            preds.append(model(x).item())

    preds = np.array(preds, dtype=float)
    true_vals = np.array(true_vals, dtype=float)

    mse = float(np.mean((preds - true_vals) ** 2))

    if np.std(preds) == 0.0 or np.std(true_vals) == 0.0:
        corr = 0.0
    else:
        corr = float(np.corrcoef(preds, true_vals)[0, 1])

    return mse, corr
